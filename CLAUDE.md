## 파일 구조

```
yeoyu-site/
├── index.html        ← 메인 사이트 (한 파일에 HTML+CSS+JS 모두 들어 있음)
├── images/           ← 사이트에 쓰는 사진들
│   ├── hero.jpg              (메인 화면 큰 사진)
│   ├── interior-living.jpg   (거실)
│   ├── interior-loft.jpg     (다락방)
│   └── gallery-01~05.jpg     (갤러리 5장)
├── cards/            ← 사용설명서 카드뉴스 8장 (HTML 한 장씩)
│   ├── card_00.html ~ card_07.html
│   └── style.css             (카드뉴스 공용 스타일)
├── .wrangler/        ← Cloudflare 배포 캐시 (자동 생성, 건드리지 말 것)
├── .git/             ← git 내부 폴더 (건드리지 말 것)
├── PLAN.md           ← 사이트 기획서 (요금/시설 정보 원본)
├── README.md         ← 담당자(사람)용 안내
└── CLAUDE.md         ← Claude(AI)용 작업 규칙 (이 파일)
```

## 작업 환경 확인 (담당자가 처음 시작할 때)

담당자가 자기 컴퓨터에서 처음 작업한다면 먼저 아래를 확인:

1. **레포 클론 위치**: 보통 `~/yeoyu-site/` 같은 곳에 클론되어 있음. 없으면:
   ```bash
   git clone https://github.com/haeory-cyber/yeoyu-site.git
   cd yeoyu-site
   ```
2. **현재 디렉토리 확인**: `pwd` 했을 때 yeoyu-site 안에 있어야 함
3. **git 상태 확인**: `git status` — 깔끔한지, 변경사항 있는지

### 1-구) 레포에 박힌 정적 이미지 (안전망)
`images/*.jpg`는 Supabase 장애·조회 실패 시 백업 표시용으로 유지.
일반적으로 건드릴 일 없음. 교체하려면 파일명 맞춰 덮어쓰고 push.

### 2-1) 사진 용량/크기 가이드 (중요)
- **권장 용량**: 한 장당 500KB 이하 (최대 1MB)
- **권장 가로 폭**: 1600px 이하 (hero.jpg는 1920px까지 OK)
- **포맷**: jpg 권장 (png는 용량이 큼)
- 스마트폰 원본(5~10MB)을 그대로 올리면 사이트가 느려지므로 압축 필요
- 압축 도구: https://squoosh.app (브라우저에서 바로 가능)
- 압축 후 파일 이름은 반드시 원래대로 (`hero.jpg`, `gallery-01.jpg` 등)

### 3) 요금 변경
```
index.html에서 '360,000원', '250,000원', '20,000원' 검색해서 수정.
JS 자동 계산 부분도 같이 수정해야 함:
  - PRICE_WEEKEND, PRICE_WEEKDAY, PRICE_EXTRA 변수
  - 요금표 섹션의 표시 텍스트
세 곳 모두 일치시켜야 한다.
```

### 4) 텍스트(소개글, 카피) 수정
```
index.html에서 해당 문구 검색 후 직접 수정.
한글 깨짐 방지: 파일을 UTF-8로 저장.
```

### 5) 연락처/계좌 변경
```
index.html에서 '010-4342-9712'와 '3333-37-0993100' 검색.
PLAN.md, CLAUDE.md(이 파일)에도 같은 정보가 있으니 동시 수정.
```

### 6) 예약폼 항목 추가/변경
```
index.html의 <form id="reserve-form"> 섹션 수정.
지금은 데이터를 어디로 보내는지 미정 상태(TODO 주석).
이메일로 받고 싶으면 Formspree 연동, 카카오/텔레그램은 봇 연동 필요.
이건 후니님과 상의 후 진행할 것.
```

### 7) 사용설명서 카드뉴스(`cards/`) 수정
```
카드뉴스는 cards/card_00.html ~ card_07.html 8장으로 구성.
- 한 장의 텍스트만 바꿀 때: 해당 card_XX.html 파일 열어서 직접 수정
- 색·폰트·레이아웃 공통 변경: cards/style.css 수정
- 새 카드 추가: card_08.html 만들고 index.html에서 어떻게 노출할지 담당자와 협의
- 각 카드를 미리 보려면: 브라우저에서 cards/card_00.html 열어보기
```

### 8) 변경 사항 배포
```bash
git status              # 무엇이 바뀌었는지 먼저 확인
git add .
git commit -m "수정 내용 요약 (예: 메인 사진 교체)"
git push
# 1~2분 후 https://yeoyu.poomasi.org 에 자동 반영
```

## 절대 하지 말 것

- **🔴 main 브랜치에 검증 안 한 채로 push 금지** — 로컬에서 브라우저로 한 번 열어보고 깨진 곳 없는지 확인 후 push
- **🔴 .git 폴더 삭제 금지**
- **🔴 GitHub 토큰/비밀번호를 코드에 넣지 말 것** — 만약 폼 연동 등으로 키가 필요하면 후니님께 문의
- **🔴 cards/ 폴더의 카드뉴스는 디자인 톤이 정해져 있음** — 마음대로 색·폰트 바꾸지 말 것 (세이지 그린 #8FB8AD + Noto Serif/Sans KR)
- **🔴 사이트 정체성 변경 금지** — "농어촌민박", "공유별장" 컨셉은 유지

## 도움 요청

작업이 막히거나 잘 모르겠으면 사용자(담당자)에게 멈추고 물어볼 것.
혼자 추측해서 진행하지 말고, 절대 거짓말로 둘러대지 말 것.

후니님(소유자) 연락처는 사용자에게 직접 물어볼 것.
담당자가 git 명령을 처음 다룬다면, 명령을 그냥 실행하기보다 한 줄씩 무엇을 하는지 설명해주면 좋다.

## 첫 작업 시 체크리스트

- [ ] `git pull` 했는가 (다른 사람이 먼저 수정했을 수 있음)
- [ ] 수정할 파일을 백업했는가 (`cp index.html index.html.bak_YYYYMMDD`)
- [ ] 사용자에게 무엇을 어떻게 바꿀지 확인했는가
- [ ] 사진을 바꾼다면 용량이 1MB 이하인가
- [ ] 수정 후 로컬에서 한 번 열어봤는가 (`python3 -m http.server` 후 브라우저)
- [ ] 깨진 곳 없이 깔끔한가 (모바일·PC 모두)
- [ ] commit 메시지를 한국어로 명확히 적었는가
- [ ] push 후 1~2분 뒤 https://yeoyu.poomasi.org 에서 확인했는가
- [ ] 백업 파일(`*.bak_*`)을 commit에 포함시키지 않았는가
